# 概要

Rust の Async-graphql で GraphQL サーバの実装を試しました。
1:N 関係の構造で何も考えずにリゾルバを実装すると N+1 問題が発生するため、Dataloader を使って解決してみました。

## スキーマ

Query で user と item 取得時
それぞれ id を渡すと一行、渡さないとすべて取得するようなものを実装します。

```graphql
type Item {
  id: ID!
  name: String!
  description: String
  user: User
}

type Query {
  user(id: ID): [User!]!
  item(id: ID): [Item!]!
}

type User {
  id: ID!
  name: String!
  description: String
  items: [Item!]
}
```

## DB 構造

1:n 関係をもつ user および item テーブルで試しました。

```sql
CREATE TABLE `user` (
  `id` int NOT NULL AUTO_INCREMENT,
  `name` varchar(255) COLLATE utf8mb4_unicode_ci NOT NULL,
  `description` text COLLATE utf8mb4_unicode_ci,
  `deleted` tinyint(1) NOT NULL DEFAULT '0',
  `created_at` timestamp NOT NULL,
  `updated_at` timestamp NOT NULL,
  PRIMARY KEY (`id`)
);

CREATE TABLE `item` (
  `id` int NOT NULL AUTO_INCREMENT,
  `user_id` int NOT NULL,
  `name` varchar(255) COLLATE utf8mb4_unicode_ci NOT NULL,
  `description` text COLLATE utf8mb4_unicode_ci,
  `deleted` tinyint(1) NOT NULL DEFAULT '0',
  `created_at` timestamp NOT NULL,
  `updated_at` timestamp NOT NULL,
  PRIMARY KEY (`id`),
  KEY `FK_item_user` (`user_id`),
  CONSTRAINT `FK_item_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`)
);
```
